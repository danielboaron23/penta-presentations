The method · /skill-creator

How a skill gets built

1

Clarify

A short, focused set of questions — never a wall. Stop once the intent is clear.

2

Study the source

Clone the linked repo for proven patterns to distill (not copy).

3

Scaffold

init_skill.py → SKILL.md + references/ + assets/.

4

Write

A lean SKILL.md (triggers + workflow) and the reusable references/assets.

5

Validate & package

quick_validate.py → package_skill.py → a shareable .skill.

6

Install

Project-scoped in .claude/skills/ of both repos, so they load automatically.

Skill 01 · Penta PM · the prompt

The original prompt

DYou typed /skill-creator
I want to be a product manager that will help me create specs or PRDs and help me think about new features and new screens I want to build in this product. It will be connected to the code structure, the design system, and everything in this repo. The main reason: help me characterize and make better decisions about the pages I want to create. Ask me questions to clarify my idea. This skill will live on this project.
https://github.com/RefoundAI/lenny-skills.git

Both repos

PRD + brief + mapping + handoff

Full PRD, 1-pager brief, DS/route mapping, and a designer handoff.

Distill lenny-skills

PM frameworks condensed into references/, with credit.

Skill 02 · Penta Design Research · the prompt

The original prompt

DYou typed /skill-creator
create design research that sees how our competitors design the features we want to build. It will connect to the Mobbin MCP and get real references. Its "brain" comes from a competitive-analyst agent. It will deliver HTML slides + attach the Mobbin references, so we can paste them into the presentation and into Figma. Ask me questions to make it better.
https://github.com/Owl-Listener/designer-skills.git

Skill 02 · Penta Design Research · the way of taking

Clarify before building

Local images + links

Save screenshots locally and keep every mobbin_url.

What to take from this

  • 1A good skill starts with a plain-language prompt + a source to learn from — not a perfect spec.
  • 2Clarify first. A few sharp questions turn an idea into a precise, useful skill.
  • 3Distill, don't copy. Condense proven frameworks into references/, with credit.
  • 4Make skills project-grounded — connected to the code, the design system, Notion, and Mobbin.
